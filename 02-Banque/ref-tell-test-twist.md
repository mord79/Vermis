# [RÉF] Tell, Test, Twist — Graverobber's Guide

## Idée centrale

Pattern de level design en trois temps emprunté au jeu vidéo : **Tell** (établir une info dans un contexte sûr — montrer comment marche la chose), **Test** (un obstacle plus facile pour qui a noté le Tell), **Twist** (même info, contexte détourné, exige un saut logique). D. G. Chapman l'adapte au TTRPG avec trois garde-fous : pas de gating dur derrière un Test, encourager les *schemes* hors structure, multiplier les routes (Jaquaying) pour absorber les Tells ratés.

## Pourquoi intéressant pour Vermis

- **Player Skill** — outil concret pour produire l'*In-World Problem Solving* (type 1 de la taxonomie [[ref-types-of-player-skill]]). Récompense l'attention diégétique, pas le min-max.
- **Arbitrages > règles** — Chapman écrit explicitement « we don't do that here » sur les jets à haut enjeu unique. Filiation directe avec le principe directeur de Vermis.
- **Procédure / sandbox** — Jaquaying et doctrine multi-routes alignés avec la sandbox grimdark.
- **Couplages naturels :**
    - [[ref-rooms-in-pairs]] — connectivité narrative (paires setup/punchline) ; TTT et Pairs peuvent s'empiler dans les mêmes salles.
    - [[ref-types-of-player-skill]] — TTT est l'outil de prep qui produit le défi que la taxonomie nomme.
    - [[ref-markov-stocking]] — la matrice Markov peut générer la séquence Tell → Test → Twist comme transitions contraintes.

## Pourquoi pas adopté maintenant

- Ce n'est pas une mécanique — c'est un outil de prep MJ, pas une règle. Rien à inscrire dans `Vermis_Rules.md`.
- Risque d'érosion grimdark si appliqué systématiquement : un donjon trop bien TTT-é devient un cours de logique prévisible. À utiliser comme heuristique parmi d'autres, pas comme grille obligatoire.
- Concept simple — Chapman lui-même reconnaît qu'il flotte déjà dans la littérature OSR. Valeur surtout pédagogique pour les conseils MJ.

## Conditions de réactivation

- Lors de la rédaction du chapitre **Mégadonjon** ou **Conseils MJ** de la Bible.
- Comme grille d'analyse pour relire un donjon prep avant playtest (« où est le Tell ? le Test ? le Twist ? puis-je contourner ? »).
- Si un retour de table indique que l'attention environnementale n'est pas récompensée — TTT donne une checklist concrète.

## Variantes

- **Tell them again** — multiplier les Tells via différentes routes (variante explicite de Chapman pour absorber le Jaquaying).
- **Layer TTT** — empiler plusieurs structures TTT en parallèle dans un même secteur (multi-routes = multi-Tells).
- **Multi-use Tell** — un même indice débloque plusieurs Tests/Twists différents (économie d'éléments, lit l'agentivité).
